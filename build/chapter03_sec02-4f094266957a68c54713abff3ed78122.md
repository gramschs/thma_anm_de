## Lösen mit `np.linalg.solve`

Wir starten sofort mit dem Code. Das Obstmarkt-System aus Kapitel 3.1 kennen
wir bereits:

```{code-cell} python
# Koeffizientenmatrix: Zeile = Einkaufstag, Spalte = Fruchtsorte
A = np.array([
    [3, 2, 1],
    [2, 3, 0],
    [1, 1, 3],
], dtype=float)

# Rechte Seite: bezahlte Gesamtbetraege in Euro
b = np.array([2.10, 1.70, 2.80])

# Eine einzige Zeile loest das gesamte System
x = np.linalg.solve(A, b)

# x[0] = Preis Apfel, x[1] = Preis Banane, x[2] = Preis Orange
print(f'Preis Apfel:  {x[0]:.2f} €')
print(f'Preis Banane: {x[1]:.2f} €')
print(f'Preis Orange: {x[2]:.2f} €')
```

`np.linalg.solve(A, b)` loest $\mathbf{A} \cdot \vec{x} = \vec{b}$ direkt.
Intern nutzt NumPy dafür eine sogenannte **LU-Zerlegung**, die die Matrix in
zwei Dreiecksmatrizen zerlegt. Das ist schneller und genauer als das
Gausssche Eliminationsverfahren von Hand.

+++

## Das Ergebnis immer prüfen: die Probe

*Wie wissen wir, dass das Ergebnis stimmt?* Wenn $\vec{x}$ die richtige Lösung
ist, muss das Matrixprodukt $\mathbf{A} \cdot \vec{x}$ exakt den Vektor
$\vec{b}$ ergeben. Das ist unsere **Probe**:

```{code-cell} python
# Probe: A mal x muss gleich b sein
b_probe = A @ x   # @ ist der Matrixmultiplikations-Operator in NumPy

print('Ergebnis A @ x:', b_probe)
print('Soll-Wert   b: ', b)

# np.allclose prueft, ob beide Arrays bis auf Rundungsfehler gleich sind
# (besser als == , weil Fliesskommazahlen nie exakt uebereinstimmen)
print('Probe bestanden:', np.allclose(b_probe, b))
```

Die Probe sollte in jedem Programm stehen. Ein einzelner Tippfehler in der
Matrix kann sonst unbemerkt ein falsches Ergebnis liefern.

+++

## Die Inverse: eine Alternative für Spezialfälle

Es gibt einen zweiten Weg zur Lösung. Aus
$\mathbf{A} \cdot \vec{x} = \vec{b}$ folgt durch Multiplikation mit der
**inversen Matrix** $\mathbf{A}^{-1}$ von links:

$$\vec{x} = \mathbf{A}^{-1} \cdot \vec{b}$$

Die Inverse ist so definiert, dass $\mathbf{A}^{-1} \cdot \mathbf{A}$
die **Einheitsmatrix** $\mathbf{I}$ ergibt (das Matrix-Äquivalent der Zahl 1):

```{code-cell} python
# Inverse berechnen
A_inv = np.linalg.inv(A)

# Probe: A_inv @ A soll die Einheitsmatrix sein
# np.round rundet auf 10 Dezimalstellen, damit Rundungsfehler nicht stoeren
print('A_inv @ A (soll Einheitsmatrix sein):')
print(np.round(A_inv @ A, 10))

# Loesung ueber die Inverse
x_via_inv = A_inv @ b
print('Loesung via Inverse:', x_via_inv)
```

```{admonition} solve oder inv?
:class: note
**Normalfall: `np.linalg.solve`.**
Es ist schneller und numerisch stabiler als `inv`, weil es die Matrix
nicht explizit invertiert.

**Ausnahme: `np.linalg.inv`** lohnt sich, wenn man dasselbe System
$\mathbf{A} \cdot \vec{x} = \vec{b}$ für viele verschiedene rechte Seiten
$\vec{b}$ lösen muss, die Matrix $\mathbf{A}$ aber immer gleich bleibt.
Dann berechnet man $\mathbf{A}^{-1}$ einmal und multipliziert danach nur
noch.
```

import numpy as np

A = np.array([

## Zusammenfassung und Ausblick

`np.linalg.solve(A, b)` löst ein LGS in einer Zeile. Die Probe
`np.allclose(A @ x, b)` sichert jedes Ergebnis ab. Die Inverse
`np.linalg.inv(A)` ist eine Alternative, wenn dieselbe Matrix mehrfach
für verschiedene rechte Seiten verwendet wird.

Im nächsten Kapitel verlassen wir den Obststand und wenden das Gelernte
auf ein reales Ingenieurproblem an: die Berechnung von Temperaturen in einer
mehrschichtigen Wand. Das Prinzip bleibt identisch, aber die Gleichungen
kommen jetzt aus der Physik.
